# View Plus

View Plus 是一个 Obsidian 桌面端插件，专注于三件事：

- 用内置视图打开非 Markdown 文本文件
- 为常见代码/配置文件提供精简后的基础语法高亮与编辑保存
- 随时用系统默认应用打开当前文件

它不再接管隐藏文件、目录扫描或文件树补丁逻辑，定位就是一个轻量的非 Markdown 文件查看/编辑增强插件。

> **仅限桌面端。** 系统默认应用打开能力依赖桌面环境中的 Electron API。

## 功能一览

| 功能 | 说明 |
| --- | --- |
| **文本文件编辑** | 以独立视图打开常见代码、配置和纯文本文件，编辑后自动触发保存 |
| **语法高亮** | 仅保留最常用的一组高亮语言包，其他文本类型仍可正常打开和编辑 |
| **媒体预览** | 内嵌预览 Obsidian 默认不直接处理的部分图片、音频、视频格式 |
| **系统应用打开** | 支持双击、右键菜单、命令面板、视图工具栏按钮调起系统默认应用 |
| **大文件保护** | 文本内容超过 5 MB 时不直接载入编辑器，避免卡顿 |

## 安装

### 社区插件市场（推荐）

1. 打开 **设置 → 第三方插件 → 浏览**
2. 搜索 **View Plus**
3. 点击 **安装**，再点击 **启用**

### 手动安装

1. 从 [最新 Release](../../releases/latest) 下载 `main.js`、`manifest.json`、`styles.css`
2. 复制到 `<vault>/.obsidian/plugins/obsidian-view-plus/`
3. 重启 Obsidian，在 **设置 → 第三方插件** 中启用

## 用系统应用打开

对任意非 Markdown 文件，有四种方式调起系统默认应用：

- **双击**文件浏览器中的文件名
- **右键** → "Open with system app"
- 在文件查看器视图右上角点击**外部链接按钮**
- **命令面板** → "View Plus: Open with system app"（可绑定快捷键）

## 媒体查看器

以下格式会在 Obsidian 内嵌预览：

| 类型 | 格式 |
| --- | --- |
| 图片 | `avif` `ico` `tiff` `tif` |
| 音频 | `aac` `opus` |
| 视频 | `mov` `avi` `mkv` `wmv` `m4v` |

浏览器无法解码时会显示提示，并引导使用外部应用打开。

## 内置文本查看器

文本/代码文件会用 CodeMirror 视图打开，提供：

- 精简后的基础语法高亮
- 行号
- 当前行高亮
- 自动保存触发
- 超过 **5 MB** 时的安全提示

当前保留高亮的常见格式：

| 分类 | 扩展名 |
| --- | --- |
| Web / 脚本 | `js` `mjs` `cjs` `ts` `jsx` `tsx` `py` |
| 系统语言 | `c` `h` `cpp` `hpp` |
| 配置 / 数据 | `json` `yaml` `yml` |
| 样式 / 标记 | `css` `scss` `sass` `less` `html` `htm` |

其他仍会由插件接管的文本扩展名，会继续以纯文本方式打开和编辑，但不再额外加载对应语言高亮包。

## 设置

当前版本没有额外功能开关；启用插件后即可使用。

## 开发

```bash
npm install          # 安装依赖
npm run dev          # 监听模式（增量构建）
npm run build        # 生产构建
npm run deploy       # 构建并部署到本地 vault
```

TypeScript 源码位于 `src/`：

| 文件 | 职责 |
| --- | --- |
| `src/main.ts` | 插件入口、视图注册、外部打开入口 |
| `src/viewer.ts` | 文本查看器、媒体查看器、语法高亮、系统打开入口 |
